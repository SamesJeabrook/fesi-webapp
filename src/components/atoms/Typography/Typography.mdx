## Available HTML Elements (`as` prop)

### Headings
- `h1`, `h2`, `h3`, `h4`, `h5`, `h6`

### Text Elements
- `p` (default), `span`, `div`

### Semantic Elements
- `label`, `legend`, `figcaption`

### Text Formatting
- `small`, `strong`, `em`, `mark`, `del`, `ins`, `sub`, `sup`

## Usage Examples

```tsx
import { Typography } from '@/components/atoms';

// Basic usage (renders as <p>)
<Typography variant="body-medium">
  Regular paragraph text
</Typography>

// Semantic heading with custom style
<Typography variant="heading-2" as="h1">
  Page Title
</Typography>

// Form label with overline style
<Typography variant="overline" as="label">
  FIELD LABEL
</Typography>

// Caption text
<Typography variant="caption" as="figcaption">
  Image caption below a figure
</Typography>

// Custom styling
<Typography 
  variant="heading-3" 
  as="h2" 
  className="custom-class"
  style={{ color: 'blue' }}
>
  Custom styled heading
</Typography>
```

## Best Practices

### âś… Do's
- Use semantic HTML elements that match the content purpose
- Choose typography variants that reflect content hierarchy
- Use `heading-1` through `heading-8` for actual headings
- Use `body-*` variants for paragraph content
- Use `caption` for image captions and secondary information
- Use `overline` for labels and category indicators

### âťŚ Don'ts
- Don't use heading variants for non-heading content just for styling
- Don't skip heading levels in your document structure
- Don't use typography variants that conflict with semantic meaning
- Don't apply additional font-size styles that override the design system

## Accessibility

The Typography component promotes accessible markup by:

- **Semantic HTML**: Encourages proper use of heading hierarchy
- **Consistent styling**: Maintains visual hierarchy that matches semantic structure
- **Flexible elements**: Allows choosing the right HTML element for the content
- **Screen reader friendly**: Proper heading structure aids navigation
